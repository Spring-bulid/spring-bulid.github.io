🔧 LKM 模式详解

LKM（Loadable Kernel Module）是 KinSU 的核心冒险模式。它以可加载内核模块的方式注入系统，无需重新编译内核即可获得 Root 权限。

  • 无需修改内核源码，刷入即用
  • 支持动态加载与卸载，灵活切换
  • 兼容非 GKI 设备，冒险范围更广
  • 与 GKI 模式自动切换，无缝衔接

🌍 GKI 设备适配

对于 GKI（Generic Kernel Image）设备，KinSU 通过 init_boot 分区检测自动识别，并提供完整的内核级 Root 支持。

  • 自动检测 /dev/block/by-name/init_boot 分区
  • 刷入第三方内核后仍可正确识别 GKI 设备
  • OTA 更新后自动切换活跃槽位，Root 不丢失
  • 支持 KPM 和 SuSFS 高级功能

🧩 模块系统

模块是 KinSU 冒险者的魔法道具。每个模块都能扩展系统功能，从系统美化到性能优化，无所不能。

  • 支持 WebUI 可视化配置界面
  • 模块仓库一键安装，自动更新检测
  • 安全沙箱隔离，模块互不干扰
  • 支持模块卸载与禁用，随时切换

🛡 超级用户管理

精细的权限控制系统，为每个应用独立配置 Root 权限。支持模板系统，一键应用常用权限配置。

  • 应用级 Root 权限开关
  • 自定义 SELinux 上下文与能力集
  • 模板系统：保存、分享、复用权限配置
  • 操作日志记录，一切尽在掌握

🕵 SuSFS 隐身术

SuSFS（Suspend FS）是 KinSU 的隐身结界，可以隐藏内核补丁和用户空间模块的存在，让 Root 状态遁入无形。

  • 隐藏内核模块加载状态
  • 隐藏用户空间模块文件
  • 对抗检测工具的扫描
  • 仅 GKI 设备可用（需内核支持）

⚙ KPM 内核补丁

KPM（Kernel Patch Module）是 KinSU 的高级内核补丁系统，可以在运行时动态修补内核行为，实现更深层的能力扩展。

  • 运行时内核补丁注入
  • 支持自定义补丁模块
  • 与 LKM 模式协同工作
  • 仅 GKI 设备可用（需内核支持）

🎨 主题幻化

KinSU 内置强大的主题引擎，支持动态取色、多种配色风格和字体选择，让你的管理器独一无二。

新手冒险指南

从零开始的 Root 冒险，只需简单几步即可踏上旅途

1

解锁 Bootloader

首先需要解锁设备的 Bootloader，这是通往异世界的入口。不同厂商的解锁方式各异，请查阅设备对应教程。

2

刷入 KinSU 内核

下载对应设备的 KinSU 内核镜像，通过 fastboot flash boot 或 fastboot flash init_boot 刷入设备。

3

安装 KinSU 管理器

从 GitHub Releases 下载最新版管理器 APK 并安装，这是你冒险的指挥中心。

4

开启冒险

打开管理器，确认状态显示「工作中」，即可开始安装模块、授权应用，享受 Root 的力量！
